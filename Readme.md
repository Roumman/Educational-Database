## О проекте

Учебная база данных, моделирующая основные процессы банка:
* управление клиентами и счетами;
* операции с депозитами и кредитами;
* транзакции и выписки;
* бизнес‑правила через триггеры;
* система учебных заданий.

**Цель:** дать практический опыт работы с SQL на примере банковской сферы.

## Особенности

* **Реалистичная схема БД** — отражает настоящие банковские процессы.
* **Автоматизированная бизнес‑логика** через триггеры SQLite.
* **Система учебных заданий** с проверкой решений.
* **Готовые тестовые данные** для начала работы.
* **Примеры запросов** разного уровня сложности.

## Структура базы данных

Основные таблицы:
* `client` — информация о клиентах банка;
* `account` — банковские счета;
* `loan` — кредиты;
* `deposit` — депозиты;
* `account_transaction` — транзакции по счетам;
* `payment_schedule` — графики платежей;
* `bank_branch` — филиалы банка;
* `employee` — сотрудники.

## Установка и запуск

### Предварительные требования
* Python 3.8+
* SQLite 3.30+

### Шаги установки

1. Клонируйте репозиторий:
```bash
git clone https://github.com/Roumman/Graduation-project
```
2. Создайте и заполните базу данных:
```bash
python scripts/populate_db.py
```
## Использование

### Для обучения SQL

- Откройте `database/create_db.sql` для изучения схемы БД.
- Выполните запросы из `examples/basic_queries.sql`.
- Попробуйте решить задания из `exercises/exercises.json`.

### Для практики с триггерами

- Изучите логику в `database/triggers.sql`.
- Выполните операции, которые активируют триггеры:
  - добавьте транзакцию — проверьте автоматическое обновление баланса;
  - создайте кредит — убедитесь в формировании графика платежей.

### Для преподавателей

Используйте `exercises/` для создания собственных заданий. Формат JSON позволяет легко добавлять новые упражнения.

## Учебные задания

Платформа включает задания трёх уровней:

### Базовый

- простые `SELECT`‑запросы;
- `INSERT` для добавления данных;
- `UPDATE` для изменения записей.

### Продвинутый

- `JOIN` между таблицами;
- агрегатные функции (`SUM`, `COUNT`);
- подзапросы.

### Экспертный

- написание сложных триггеров;
- оптимизация запросов;
- анализ данных.
